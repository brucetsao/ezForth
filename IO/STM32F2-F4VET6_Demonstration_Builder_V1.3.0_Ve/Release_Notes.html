Update history

The STM32 Demonstration Builder platform is a completely new way to deliver a demonstration that can also be fully or partially reused in real applications. It relies on a full set of software components, coming in flexible licensing schemes allowing easy reuse and redistribution. All these components are organized within a module architecture that allows them to be reused separately in standalone applications.

The versatility of the STM32 Demonstration Builder platform allows the dynamic addition of modules, granting access to common resources, including storage, graphical components and widgets, and memory management.
The STM32 Demonstration Builder platform is built around the STM32 graphical library and the FreeRTOS real-time operating system, and uses almost the entire STM32 capability to offer a large scope of usage.

This Demonstration supports STM32F2xx and STM32F4xx devices and runs on STM3220G-EVAL, STM3240G-EVAL and STM324x7I-EVAL evaluation boards from STMicroelectronics.

For more details, please refer to the user manuals "UM1550 STM32 Demonstration Builder developer guide" and "UM1549 STM32 Demonstration Builder", available for download from the STMicroelectronics microcontrollers website: www.st.com/STM32

  • RealView Microcontroller Development Kit (MDK-ARM) toolchain V4.50 + ULINK
    • A software patch is needed to use STM32F427x/437x devices with MDK-ARM V4.50 or later. If you do not have this patch installed, you can install it from the following folder "\Utilities\Third_Party\Toolchains_STM32F4x7II_Patches\MDK-ARM_STM32F4x7II_Support.zip"

Supported Devices and EVAL boards

  • STM32F20x/21x devices and STM322xG-EVAL board
  • STM32F40x/41x devices and STM324xG-EVAL board
  • STM32F427x/437x devices and STM324x7I-EVAL board

Known Limitations

  • No TrueSTUDIO project for STM32F427x/437x devices
  • Audio recorder module: when using uSD card as media storage, after minutes of recording the demonstration may hangs
  • Wav files with sample rate 32 KHz and 48 KHz are not supported

V1.0.1 / 06-September-2012

Main Changes

  • Fix V1.0.0 release limitations:
    • uSD card I/O operations limitation
      • Description
        • The uSD card (through SDIO) I/O operations could be randomly altered during the logical read operations through the file system, this can be seen by detecting the presence of some noises when playing some wav files or showing BMP image from the uSD.
      • Fix details
        • fatfs_drv.c file: add a local scratch buffer and align the I/O buffer in the read and write operations when input buffer is not 32-bits aligned.
    • File browser limitation
      • Description
        • Open audio player and mix a playlist from USB disk and SD card then unplug the current used disk => No update on folders tree after audio player close.
      • Fix details
        • mod_audio.c, mod_recorder.c, mod_camera.c and mod_image.c files: remove the reset of the FILEMGR_CurrentDirectory buffer before the call of the FILMGR_DirectEx().
        • mod_filemgr.c file: manage recursively the FILMGR_DirectEx() resources to keep the current FILEMGR_CurrentDirectory value for each instance.

Known Limitations

  • On some EVAL boards the uSD card (through SDIO) I/O operations could be randomly altered during the logical read operations through the file system, this can be seen by detecting the presence of some noises when playing some wav files or showing BMP image from the uSD. This phenomena is mainly due to the presence of noise on the uSD data lines coming from other devices running in high speed on the board. The issue will be fixed definitively in the next releases.
